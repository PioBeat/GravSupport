0.6.0-alpha

  • Added
    • After 'Grav Create Theme' action completes, the project file explorer jumps to the freshly created theme directory
    • Tool window: Navigate directly to Grav's system/config/system.yaml file from the tool window via a link
    • Project settings page: Grav version is now shown for the current project
    • Plugin is able to convert a previously created non-Grav project type to a Grav project
  • Changed
    • Now using *.svg icons
    • Uniform icon usage for file explorer, tool window, project wizard, etc.
    • Minor UI changes in Grav settings
    • 'Grav Create Theme' Action can now be called when indexing
    • Module type is set correctly now to GRAV_TYPE in PhpStorm. For IU this was set automatically.
    • PhpStorm: Project icon is now visible (same as in IU)
  • Internal Changes and Bugfixes
    • Platform Update to branch number 203.*, requiring now Java 11
    • Compatibility fixes (e.g., error reporting)
    • Many bug fixes for the Grav Language Editor
    • Fixed: Grav version in project wizard were not downloaded / processed correctly
    • file type pattern *.yaml registration for blueprints/theme configuration fixed (no ambiguity anymore)

0.4.2

  • Added
    • Automatic error reporting feature implemented. All plugin's exceptions that are occurring within IntelliJ IDE can now be reported within the IDE. The issues will be added to a specific repository (https://github.com/Grav-IntelliJ-Plugin/GravSupport-Plugin-BugTracker) dedicated for automatic error reports. Additional user requests can be still submitted to the original issue tracker (https://github.com/PioBeat/GravSupport).
    • Improved notifications: Warnings are shown when PHP version does not match the Grav requirements of the current Grav installation
  • Changed
    • *Create New Grav Project* wizard updated. Layout is more clear now.
  • Bugfix
    • Fixed some bugs concerning the detection whether Grav's devtool plugin is being installed correctly or not.
    • "Create Theme" action should work now correctly again. Moreover, folder is refreshed afterwards. Prior to that, the folder has to be manually synchronized.
    • Bugfixes in the *Create New Grav Project* wizard
